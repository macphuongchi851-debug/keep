# Day 2：启动、状态与基础排障 —— 先确认机器活着，再谈学习

## 今天为什么学这个

昨天你已经知道 OpenClaw 大概由什么组成。

今天开始落地：
- Gateway 到底有没有运行
- Control UI 能不能打开
- status 里该看什么
- 出问题先从哪排

这节的目标很朴素：**别一上来就在死人身上做心肺复苏式学习。**

## 今天的目标

你需要做到：

1. 会用 `openclaw status` 看系统状态
2. 会启动或重启 Gateway
3. 会打开 Control UI
4. 知道最基础的排障顺序

---

## 一、先跑状态命令

执行：

```bash
openclaw status
```

你重点观察：
- Gateway 是否在运行
- 有没有明显报错
- Control UI 地址是否清晰
- 当前整体是否处于可用状态

如果想看更多细节，可以后面再看更深的状态，但今天先把基础看懂。

---

## 二、如果没起来，就启动 Gateway

```bash
openclaw gateway start
```

启动后，再执行一遍：

```bash
openclaw status
```

如果已经在运行，也可以记一下重启命令：

```bash
openclaw gateway restart
```

重点不是背命令，而是形成习惯：
**先看状态，再决定要不要启动/重启。**

---

## 三、打开 Control UI

通常在浏览器打开：

```text
http://127.0.0.1:18789
```

如果你环境改过端口，就以 `openclaw status` 里显示的地址为准。

打开后，试着发一句：

```text
你好，我在做 Day 2 的连通性测试。
```

今天先不讲回答质量，重点只是确认：
- 页面能打开
- 消息能发
- 回应能回来

---

## 四、学一个最基础排障顺序

以后别一出问题就瞎猜，按这个顺序来：

### 1）先看 `openclaw status`
先确认是不是整体就没活着。

### 2）再看 Gateway 是否需要启动/重启
如果后台服务有问题，前端再漂亮也没用。

### 3）再看 Control UI 是否能打开
如果 UI 打不开，先别怪模型。

### 4）最后再去想配置、模型、渠道问题
排障要先排“系统层”，再排“功能层”。

你做完后，要能告诉我：

1. `openclaw status` 显示的关键结果是什么
2. Gateway 当前是否在运行
3. 你是否成功打开了 Control UI
4. 如果今天 OpenClaw 突然不回话，你的第一步会做什么

---

## 常见坑

### 1）UI 打不开就怪模型
不对，先看 Gateway 和状态。

### 2）一出问题就重装
很多问题根本没到重装那一步，先看状态和日志。

### 3）没确认是否真的起来，就继续折腾后面的课
这就像地基没浇好先盖楼，纯找事。
