# Day 6：配置安全与会话边界 —— 很多坑不是功能坑，是边界坑

## 今天为什么学这个

配置文件最麻烦的，不是字段多，而是**边界感**。

尤其是：
- 多用户 DM 会不会串上下文
- 不同渠道消息是不是会混到一起
- 哪些配置一改就可能影响隐私和安全

今天就专门讲这个。

## 今天的目标

你需要理解：

1. `session.dmScope` 是干嘛的
2. 为什么多用户场景下上下文隔离很重要
3. 哪些配置属于高风险区
4. 为什么不能拿“能跑”当“安全”

---

## 一、先看 `session.dmScope`

先执行：

```bash
openclaw config get session.dmScope
```

这个字段决定：
- 私聊消息是共用主会话
- 还是按用户/渠道拆开

如果多个人都能给 OpenClaw 发私聊，而你又没做好隔离，就可能出现上下文串线。

---

## 二、什么叫“上下文串线”

比如：
- A 跟 OpenClaw 说了私人内容
- B 来问“我们刚才在聊什么”
- 如果 DM 共用上下文，模型就可能把 A 的内容带出来

这就不是小 bug 了，这是隐私事故。

---

## 三、哪些配置最值得敬畏

### 1. `session.*`
影响上下文边界、连续性和隔离。

### 2. `channels.*`
影响谁能找你、哪些群能触发、allowlist 是否合理。

### 3. `browser.*`
影响浏览器控制方式、隔离性、执行环境。

### 4. `agents.defaults.*`
影响默认 agent 的模型、工具、工作区、节奏等。

---

## 四、今天重点不是改，而是建立警惕

你今天只需要做两件事：

1. 读 `session.dmScope`
2. 用自己的话说明：为什么多用户环境下必须重视会话隔离

如果你现在就是单人环境，也要懂这个概念，因为以后很容易踩坑。
