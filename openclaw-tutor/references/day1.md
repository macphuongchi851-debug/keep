# Day 1：OpenClaw 是什么 —— 先把地图装进脑子里

## 先讲人话：你今天到底在学什么

今天不是学命令，也不是学花活。
你今天要搞明白的是：**OpenClaw 到底是个什么东西，它由哪些部分组成，你以后每一步是在操作哪一层。**

很多人一上来就想接 Telegram、切模型、搞自动化，最后学成一锅粥。根因不是手不行，是**脑子里没有地图**。

所以 Day 1 只做一件事：
**先把地图装进去。**

---

## 这东西到底是什么

你可以先把 OpenClaw 理解成：

> 一个能调用模型、会用工具、能读写文件、还能接入聊天渠道和自动化任务的个人助手系统。

它不是单纯的网页聊天框。
它更像一个带大脑、带手脚、还能接不同入口的助手框架。

### 你现在先记住 4 个核心词

#### 1. Gateway
可以把它理解成“总控中枢”。
很多能力都要经过它：消息收发、工具调用、会话管理、状态检查。

#### 2. Session
就是一次持续对话的上下文。
不是你发一句它回一句这么简单，而是“这一段连续聊天”本身。

#### 3. Workspace
就是它默认干活的工作区。
读文件、写文件、改文件，默认都围着这个目录转。

#### 4. Control UI
可以把它理解成控制面板。
你可以在里面看状态、看会话、做一些管理动作。

---

## 为什么先学这个

因为后面你会不断看到这些场景：
- 为什么网页能开，但助手不回消息
- 为什么 Telegram 接上了，但文件改不到
- 为什么模型切了，看起来又像没切
- 为什么同样一句话，在不同地方表现不一样

如果你不知道自己是在碰 Gateway、Session 还是 Workspace，排障就会像摸黑找裤腰带。

---

## 今天你真正要学会什么

学完今天，你至少要能用自己的话说清：

1. OpenClaw 不只是聊天界面
2. Gateway、Session、Workspace、Control UI 各干什么
3. 以后“接消息”“改文件”“看状态”分别大概落在哪一层

---

## 今天怎么学：按这个顺序来

### 第一步：先把 OpenClaw 想象成一家小公司

你可以这么记：

- **Gateway** = 前台 + 总控
- **Session** = 某个正在进行的客户对话
- **Workspace** = 办公桌 / 文件柜
- **Control UI** = 管理后台

这样记虽然不严谨，但对刚入门很有用。
先把感觉建立起来，比死背定义强多了。

### 第二步：看看你本机现在的工作区长什么样

先执行：

```bash
pwd
ls
```

你先别追求看懂所有文件，只要建立一个感觉：
**OpenClaw 不是飘在空中的，它是落在一个真实目录里的。**

### 第三步：看一眼系统状态

执行：

```bash
openclaw status
```

你现在不要求完全看懂输出。
你今天只需要观察：
- 它会不会返回系统信息
- 有没有 Gateway 相关状态
- 有没有模型 / channel / session 之类的线索

重点不是背输出，而是知道：
**以后很多排障，第一眼就先看 status。**

---

## 你应该得到什么感觉

如果今天学对了，你脑子里应该出现这种感觉：

> “哦，OpenClaw 不是一个页面，而是一整套系统。聊天只是入口，底下还有中枢、会话、文件区和控制面板。”

只要你有这个感觉，Day 1 就算没白学。

---

## 最容易踩的坑

### 坑 1：把 OpenClaw 当成“网页版 ChatGPT”
这会导致你后面一碰到文件、渠道、自动化就懵。

### 坑 2：一上来想把所有名词背全
没必要。先记住大块结构，再慢慢细化。

### 坑 3：只看概念，不碰状态和目录
不落地看一眼，你脑子里的地图会很虚。
