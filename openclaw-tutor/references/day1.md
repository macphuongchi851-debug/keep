## 一、先认识 4 个核心词

### 1. Gateway
Gateway 是 OpenClaw 的后台中枢。

它负责：
- 接收和路由消息
- 维护会话状态
- 管理渠道连接（Telegram、Discord 等）
- 驱动工具、定时任务、节点、浏览器等能力

你可以把它理解成：**总控后台**。

### 2. Control UI
Control UI 是网页控制台。

它的作用是：
- 展示状态
- 让你发消息
- 管理配置和观察系统

它不是 OpenClaw 的本体，而是一个**操作界面**。

### 3. Workspace
Workspace 是 OpenClaw 工作和存放记忆的默认目录。

里面通常会有：
- 人格和用户设定文件
- 记忆文件
- skills
- 项目文件

你可以把它理解成：**代理的工作台 + 外置大脑**。

### 4. Session
Session 就是会话上下文。

它决定：
- 哪些消息算同一段对话
- 什么内容会连续记住
- 不同用户/不同渠道的上下文是否隔离

你可以把它理解成：**对话轨迹容器**。

---

## 二、把关系连起来

你可以先这么理解：

- **Gateway** 是后台
- **Control UI** 是前台操作界面
- **Workspace** 是文件与记忆的落点
- **Session** 是对话上下文的边界

一句人话版总结：

> 你通过 Control UI 或 Telegram 跟 OpenClaw 说话；
> Gateway 在后台处理；
> Session 决定上下文怎么连续；
> Workspace 决定哪些东西能落成文件和长期记忆。

## 三、今天不需要干很多，只做两个动作

### 动作 1：看看你的 workspace 里有什么

```bash
cd ~/.openclaw/workspace
ls
```

如果想看完整点：

```bash
find ~/.openclaw/workspace -maxdepth 2 -type f | sort
```

### 动作 2：试着找到这些文件

你至少认一下：
- `AGENTS.md`
- `SOUL.md`
- `USER.md`
- `MEMORY.md`
- `memory/` 目录

不用死记，但要有印象：
这些文件不是摆设，它们决定 OpenClaw 之后怎么表现、怎么记东西。

## 四、今天最重要的理解

### 1）OpenClaw 不只是“模型聊天”
它是：
- 模型
- 会话系统
- 文件工作区
- 工具调用
- 渠道接入
- 自动化调度

一起构成的系统。

### 2）文件是核心现实层
很多“记忆”“设定”“行为规则”最后都要落到 workspace 文件里。

### 3）后面很多问题，本质都是边界问题
比如：
- 这是 UI 的问题，还是 Gateway 的问题？
- 这是 session 串了，还是 memory 没写？
- 这是配置错了，还是模型没切成功？

今天就是先学会区分这些层。

## 常见坑

### 1）把 UI 当本体
错。UI 只是入口，不是中枢。

### 2）把“记忆”理解成模型脑补
错。长期有用的记忆通常需要文件落地。

### 3）一上来只想学命令
命令会背，地图没装对，后面照样容易晕。
