# Day 3：模型系统入门 —— 先搞懂它怎么组织，再谈切换

## 今天为什么学这个

很多人一提模型，就只会问：
- 能不能切？
- 哪个强？
- 为什么报错？

但如果你连 OpenClaw 里模型怎么组织都没搞懂，后面切模型只会像抽奖。

今天不急着折腾切换，先把模型系统的骨架装清楚。

## 今天的目标

你需要理解：

1. 什么是 primary model
2. 什么是 fallback
3. 什么是 alias
4. 为什么“能看到模型”和“能切换模型”不是一回事

---

## 一、先看当前模型状态

执行：

```bash
openclaw models status
```

你先别急着研究每个字段，先观察：
- 当前默认模型是谁
- 是否存在 fallback
- 鉴权/可用性有没有明显异常

---

## 二、理解 3 个最关键的概念

### 1. Primary model
默认主模型。

正常情况下，OpenClaw 优先用它。

### 2. Fallbacks
备用模型。

当主模型挂了、超时了、鉴权失败了，才有可能走兜底逻辑。

### 3. Alias
别名。

因为很多模型名长得像一串咒语，alias 就是为了让调用和切换更顺手。

---

## 三、模型并不是“想切哪个就切哪个”

你要建立一个正确认识：

- 有的模型只是存在于配置/系统认知里
- 有的模型当前鉴权正常
- 有的模型虽然名字你知道，但并没在你的环境里真正可用

所以“知道一个模型名” ≠ “当前能直接切过去”。

---

## 四、今天先不折腾太多，先学会观察

你今天主要做两件事：

### 动作 1：看状态
```bash
openclaw models status
```

### 动作 2：列出模型
```bash
openclaw models list
```

你至少要记录：
- 你看到的 3 个模型名
- 当前默认模型是谁
- 有没有 fallback

---

## 五、今天的验收标准

你做完后要能回答：

1. 当前默认模型是谁
2. fallback 是干嘛的
3. alias 是干嘛的
4. 为什么“能看到模型”不等于“能切换模型”

---

## 常见坑

### 1）把 provider/model 名字记混
后面切换时很容易输错。

### 2）看到一串模型就以为全部能用
不一定。配置、鉴权、允许范围都可能限制。

### 3）没理解 fallback，就以为系统“偷偷换模型”
很多时候那不是灵异事件，是兜底机制。

---

## 做完后怎么回我

直接发我：

1. 你看到的 3 个模型
2. 当前默认模型是谁
3. 你自己解释：primary / fallback / alias 分别是什么

通过后，Day 4 我们才正式切模型。
