# Day 14：综合实战 —— 把 OpenClaw 真正用成助手

## 今天为什么学这个

前面 13 天你已经碰过：
- 架构
- 状态
- 模型
- 配置
- Telegram
- 文件
- Memory
- Skills
- Browser
- 自动化

今天不是再学新名词，而是把它们串起来，做一个真实闭环。

## 今天的目标

完成一个至少用到 3–5 项能力的小项目。

重点不是炫技，而是：
- 目标明确
- 输入明确
- 过程闭环
- 结果可验证

---

## 一、推荐项目方向

### 方案 A：每日提醒 + 课程摘要
用到：cron + 文件读取 + 对话输出

### 方案 B：网页信息抓取 + 写入记录
用到：browser + 文件写入 + workspace

### 方案 C：个人偏好型助手
用到：memory + 文件 + session 连续性

---

## 二、你必须说明 4 件事

1. 触发方式是什么
2. 输入从哪里来
3. 用到了哪些 OpenClaw 能力
4. 结果如何验证

---

## 今天的验收标准

你做完后，要能告诉我：

1. 项目目标
2. 输入
3. 过程
4. 输出
5. 验证方式

1. 项目名
