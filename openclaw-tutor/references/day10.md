# Day 10：Memory 系统 —— 让 OpenClaw 真的记住，而不是装作记住

## 先讲人话：今天在学什么

很多人以为“聊天机器人会记忆”，其实大多数时候只是**当前对话里还没忘**。

OpenClaw 真正的记忆，不靠玄学，靠文件。

今天你要学明白：

> 什么叫短期记忆，什么叫长期记忆，什么信息该记哪儿。

---

## 这套记忆系统到底是什么意思

你先记两类：

### 1）Daily memory
一般是 `memory/YYYY-MM-DD.md`

它更像日记本 / 工作日志。
适合记：
- 今天发生了什么
- 临时上下文
- 某次排障过程
- 今天刚学了什么

### 2）`MEMORY.md`
这是长期记忆。
更像“整理过的结论”。

适合记：
- 用户长期偏好
- 稳定有效的做法
- 以后还会反复用到的结论

---

## 为什么要分两层

因为不是所有信息都值得长期保存。

如果什么都往长期记忆塞，最后 `MEMORY.md` 会变成垃圾堆。
如果什么都不沉淀，那每天都像失忆重开。

所以这节课学的是判断力：

> 这条信息，是今天有用，还是以后也有用？

---

## 今天你要学会什么

1. 分清 `MEMORY.md` 和 daily memory 的区别
2. 知道什么该写长期，什么只写 daily
3. 让 OpenClaw 记住一条信息
4. 再让它把这条信息回忆出来

---

## 今天怎么学

### 第一步：选一条值得记的信息

你可以先选一条简单但真实的信息，比如：
- 你的一个固定偏好
- 你常用的工作方式
- 你之后会重复提到的一个约定

### 第二步：判断这条信息该进哪层

问自己：
- 这是今天的临时情况？
- 还是以后反复会用到？

如果只是今天的过程，记 daily。
如果是稳定偏好、长期约定，记 `MEMORY.md`。

### 第三步：写进去

重点不是“我跟你说过了你就记住”，而是：
**真正落到文件里。**

### 第四步：再测试一次回忆

过一会儿，再让它回忆这条信息。
如果它能从记忆文件里找回来，这才叫真的记住。

---

## 今天最该记住的一句话

> 没写进文件的“记住”，很多时候只是嘴上说说。

这是 OpenClaw memory 的核心现实主义。

---

## 最容易踩的坑

### 坑 1：把所有东西都写进 `MEMORY.md`
长期记忆会迅速发臭。

### 坑 2：以为对话里提过就算记住
不一定。该落盘还是得落盘。

### 坑 3：不区分“事件”和“结论”
事件更适合 daily，结论更适合长期记忆。

---

## 今天的验收标准

做完后你要能告诉我：

1. 你记录了什么信息
2. 你为什么把它记到那个位置
3. 后面是否成功回忆出来
4. 你理解的 `MEMORY.md` 和 daily memory 区别

---

## 做完后怎么回我

你直接发：

1. 你记了什么
2. 记到了哪里
3. 为什么这么分
4. 回忆测试是否成功
