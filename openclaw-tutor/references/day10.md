# Day 10：Memory 系统 —— 让 OpenClaw 真的记住，而不是装作记住

## 今天学什么

很多人以为“聊天机器人会记忆”，
其实大多数时候只是：
**当前对话里还没忘。**

OpenClaw 真正的记忆，不靠玄学，靠文件。

今天你要学明白：
- 什么叫短期 / 每日记忆
- 什么叫长期记忆
- 什么信息该记哪儿
- 为什么“想记住”不写下来，等于没记

---

## 先讲人话：这套记忆系统到底是什么意思

你先记两层：

## 1）Daily memory
一般是：

```text
memory/YYYY-MM-DD.md
```

它更像日记本 / 工作日志。
适合记：
- 今天发生了什么
- 临时上下文
- 某次排障过程
- 今天刚学了什么
- 某个阶段性决定

---

## 2）`MEMORY.md`

这是长期记忆。
更像“整理过的结论”。

适合记：
- 用户长期偏好
- 稳定有效的做法
- 以后还会反复用到的结论
- 明显值得长期保留的习惯 / 约定

---

## 为什么要分两层

因为不是所有信息都值得长期保存。

比如：
- 今天几点调了个脚本
- 某次排障试过 3 个命令

这些更适合放 daily memory。

而像：
- 用户长期偏好中文
- 某个固定链接以后都要优先发
- 某类任务要用哪套标准流程

这些就适合进 `MEMORY.md`。

简单说：
- **daily = 原始记录**
- **MEMORY.md = 提炼后的稳定结论**

---

## 为什么今天这课特别关键

因为很多人误以为 AI 会“自然记住”。
其实一旦会话变了、系统重启了、上下文没带上，
你不写下来，它就可能真忘。

所以要记住一句：

> **Text > Brain**

想记住，就写到文件里。
不写，很多时候等于没发生。

---

## 今天你要学会什么

1. 区分 daily memory 和长期记忆
2. 判断一条信息该放哪儿
3. 知道为什么“想让它记住”必须落文件
4. 学会最小记忆维护动作

---

## 今天怎么学

### 第一步：看一眼当前有哪些记忆文件

你可以先观察：
- 有没有 `memory/YYYY-MM-DD.md`
- 有没有 `MEMORY.md`
- 它们各自都记了什么类型的内容

重点不是读很多，
而是感受两者定位差异。

---

### 第二步：做一次判断练习

看下面这些信息应该放哪：

#### 情况 A
“今天修 Telegram 时发现是 allowlist 填错了用户 ID。”

更适合：
- 先写 daily memory
- 如果以后经常会踩，就再提炼进长期记忆/规则

#### 情况 B
“用户明确要求以后中文优先、回复短一点。”

更适合：
- 长期记忆 / `USER.md` / `MEMORY.md`

#### 情况 C
“今天下午做了 Day 8 课程并已通过。”

更适合：
- daily memory

---

### 第三步：做一次最小写入练习

你可以尝试记录一条今天的新信息，比如：
- 今天学了什么
- 一个刚确认的偏好
- 一条值得以后记住的结论

重点是体验这件事：
**记忆是被写出来的，不是被许愿出来的。**

---

### 第四步：学会“先记原始，再做提炼”

一个更稳的流程是：

1. 今天发生了事 → 先记进 daily memory
2. 过几天看下来，发现某些东西会反复用到
3. 再提炼进 `MEMORY.md`

这样做比“啥都往长期记忆里塞”靠谱很多。

---

## 今天最容易踩的坑

### 坑 1：把所有东西都塞进长期记忆

最后会变成垃圾堆，很难用。

### 坑 2：以为口头说“记住这个”就行

不落文件，很多时候等于没记。

### 坑 3：daily memory 记得太少

结果后面复盘时啥依据都没了。

### 坑 4：长期记忆不提炼

只会越堆越乱。
