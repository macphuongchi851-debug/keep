# Day 10：Memory 系统 —— 让 OpenClaw 真的记住，而不是装作记住

## 今天为什么学这个

OpenClaw 的“记忆”不是玄学。

核心原则就一句：
**写进文件，才算真的记住。**

今天你要把短期聊天和长期记忆彻底分清。

## 今天的目标

你需要做到：

1. 分清 `MEMORY.md` 和 `memory/YYYY-MM-DD.md`
2. 知道什么该写长期记忆，什么只写 daily
3. 让 OpenClaw 记住一条信息
4. 再让它把这条信息回忆出来

---

## 一、两层记忆怎么分

### `memory/YYYY-MM-DD.md`
当天日志，适合写：
- 今天发生了什么
- 当前项目进展
- 临时上下文

### `MEMORY.md`
长期精炼记忆，适合写：
- 用户长期偏好
- 持续有效的规则
- 稳定事实

一句话：
- daily 是流水账
- MEMORY 是精选沉淀

---

## 二、做一次“记住它”实验

找一条真的值得记的信息，比如：
- 你偏好回复更短
- 你更喜欢先给结论再给步骤
- 你固定用某个工作链接

然后让 OpenClaw 记住它。

---

## 三、再验证它有没有真记住

两种办法：

1. 看文件有没有写进去
2. 过一会儿再问它还记不记得

最好两边都确认。

---

## 今天的验收标准

你做完后要能告诉我：

1. 你让它记住了什么
2. 写进了哪个文件
3. 你怎么验证它能被回忆出来
4. `MEMORY.md` 和 daily memory 的区别

---

## 做完后怎么回我

直接发：

1. 你记住的那条内容
2. 它写到了哪里
3. 你怎么验证的
