# Day 8：Telegram 接入（下）—— 真接上、真发消息、真排故障

## 今天学什么

昨天你已经搞明白：
- Bot 是入口
- Token 是钥匙
- 访问策略决定谁能进门

今天开始真落地：

> **把 Telegram 接进 OpenClaw，让它能收你的消息、回你的消息，并且你知道出问题时怎么查。**

今天这节如果做通了，OpenClaw 才算真的从“本地能用”进化到“手机上能叫得动”。

---

## 今天你要学会的 5 件事

1. 知道 Telegram 接入要改哪些核心点
2. 能区分 token 问题、策略问题、Gateway 问题
3. 会执行重启和复查状态
4. 能在 Telegram 私聊 Bot 做实际验证
5. 如果失败，知道按顺序排查

---

## 接入 Telegram，脑子里先放这张图

```text
Telegram 私聊消息
   ↓
Bot token 认证
   ↓
Gateway 收到消息
   ↓
访问策略判断：你能不能进
   ↓
进入 session
   ↓
OpenClaw 回消息
```

所以任何接入问题，通常都落在这 3 类：

1. **接不上**：多半是 token / 认证层
2. **进不去**：多半是策略 / 授权层
3. **整体不工作**：多半是 Gateway / 系统层

这张图装脑子里，排障会清爽很多。

---

## 今天实操前的前置确认

在你真正开搞前，先确认这几件事：

- 你已经有 Telegram Bot
- 你手里有正确 token
- 你知道自己打算用 `pairing` 还是 `allowlist`
- 你当前的 OpenClaw 系统状态基本正常

如果这些都没有，别急着硬冲。
先把前置补齐。

---

## 今天怎么做：标准实操流程

## 第一步：确认当前系统状态

先执行：

```bash
openclaw status
```

为什么第一步不是改配置？
因为如果系统主干本来就不正常，
你后面 Telegram 怎么配都容易一起陪葬。

你先确认：
- Gateway 是不是正常
- 系统有没有明显致命错误

---

## 第二步：准备 Telegram 接入配置

你要落到配置里的，核心通常就这几类：

- 开启 Telegram 渠道
- 填入 bot token
- 设置私聊策略
- 如果是 allowlist，就加入允许的用户 ID

今天你不要死背某一份旧教程里的整段 JSON。
要记逻辑：

- **token 决定接不接得上**
- **策略决定让不让你进门**

也就是说：
就算 token 对了，如果你的用户 ID 不在 allowlist，
你照样可能进不来。

---

## 第三步：修改配置时的正确姿势

如果你要改核心配置文件，按这套走：

1. 先备份原配置
2. 再修改 Telegram 相关配置
3. 校验格式和字段
4. 再重启 Gateway

别偷懒。
因为 Telegram 一旦配错，
很容易出现“你以为接好了，其实系统根本没认”的情况。

---

## 第四步：重启 Gateway

配置变更后，执行：

```bash
openclaw gateway restart
```

这一步的意义是：
让新的配置真正生效。

很多人最大的低级错误就是：
**改完不重启，就开始测。**
然后测半天，纯自虐。

---

## 第五步：立刻复查状态

重启后立刻看：

```bash
openclaw status
```

你现在重点观察：
- Gateway 是否正常回来
- Telegram 相关有没有直接报错
- 有没有 unauthorized / auth / 配置异常之类的提示

这一步特别关键。
因为它能帮你判断：
问题是“系统没起来”，还是“系统起来了但 Telegram 没接上”。

---

## 第六步：去 Telegram 私聊 Bot

你发一句最简单的话就够：

```text
你好，测试一下。
```

为什么不用复杂消息？
因为今天是在测“链路通不通”，
不是测模型能不能写长文。

你现在只关心：
- 发得出去吗
- Bot 收到了吗
- OpenClaw 会回吗

---

## 如果你走的是 pairing

那你要留意：
- 是否出现批准 / 配对流程
- 你是否完成了相应审批

很多人这里会误判：
以为消息没进去就是 token 错。

其实很可能只是：
**它在等你批准。**

---

## 如果你走的是 allowlist

那你要确认：
- 当前 Telegram 用户 ID 是对的
- 你自己的 ID 已经在允许列表里

常见翻车方式：
- 加错了用户名，不是用户 ID
- 用户 ID 填错
- 改了配置没重启

---

## 出问题时，按这 3 类排

## 第一类：像 token / 认证问题

常见表现：
- unauthorized
- auth failed
- Telegram 直接连不上
- Bot 认证失败

这类优先检查：
- token 有没有填错
- 有没有复制多空格
- Bot token 是否仍有效

---

## 第二类：像访问策略问题

常见表现：
- Bot 存在
- 你也能给它发消息
- 但消息进不了实际会话
- 没有正常回复

这类优先检查：
- 当前策略是 pairing 还是 allowlist
- allowlist 里是不是你的正确用户 ID
- 是否还没完成审批/配对

---

## 第三类：像 Gateway / 系统问题

常见表现：
- `openclaw status` 本身就不正常
- 重启后状态异常
- 不只是 Telegram，别的能力也怪

这类优先检查：
- Gateway 是否成功启动
- 最近是不是改坏了别的配置
- 状态里有没有更底层报错

---

## 一个完整排障模板

你可以照这个模板判断：

### 情况 A：Telegram 明显报 unauthorized
先查 token。

### 情况 B：Bot 能收到消息，但没进入你想要的对话
先查策略 / allowlist / pairing。

### 情况 C：系统状态一片不对
先查 Gateway，别盯着 Telegram 单点发疯。

---

## 今天最容易踩的坑

### 坑 1：改完不重启

那配置压根没生效。

### 坑 2：只测消息，不看状态

这样排障会特别瞎。

### 坑 3：把策略问题误判成 token 问题

Bot 在，不代表策略就放你进来了。

### 坑 4：allowlist 填了用户名，不是用户 ID

这类错误非常常见，而且很烦人。

---
