# Day 8：Memory 不是幻觉 —— 让 OpenClaw 真正记住该记的事

## 今天为什么学这个

很多人以为“记忆”就是模型自己会记。

不是。

OpenClaw 的长期记忆核心原则非常朴素：
**写进文件，才算真的记住。**

今天这节的重点，是把短期聊天和长期记忆彻底分开。

## 今天的目标

你需要做到：

1. 区分 `MEMORY.md` 和 `memory/YYYY-MM-DD.md`
2. 知道什么该写长期记忆，什么只记 daily log
3. 亲手让 OpenClaw 记住一条信息
4. 再让它把这条信息回忆出来

---

## 一、两层记忆分别干什么

### 1. `memory/YYYY-MM-DD.md`
日记 / 当天流水账。

适合写：
- 今天发生了什么
- 刚完成什么任务
- 临时上下文
- 过程记录

### 2. `MEMORY.md`
长期精炼记忆。

适合写：
- 用户长期偏好
- 持续有效的决策
- 稳定的事实
- 以后反复会用到的信息

一句话：
- daily 是原始日志
- MEMORY 是提炼后的长期知识

---

## 二、今天做一次“记住它”实验

你可以选一条真的值得记的内容，比如：
- 你对称呼的偏好
- 你希望回复更短还是更详细
- 你某个固定工作习惯

然后直接让 OpenClaw 记住：

```text
记住：以后我更喜欢你先给结论，再给步骤。
```

如果它只是在聊天里答应，没有写文件，那不算。

---

## 三、验证它有没有真的记住

你可以用两种方式验证：

### 方式 A：直接看文件
看它有没有写入：
- `MEMORY.md`
- 或当天 `memory/YYYY-MM-DD.md`

### 方式 B：过一会儿再问
比如问：

```text
你记得我刚才让你记住什么吗？
```

真正靠谱的是：
- 它能回答出来
- 并且文件里也真的有记录

---

## 四、理解一个关键边界

不是所有东西都应该进长期记忆。

### 适合进 `MEMORY.md`
- 长期偏好
- 持续有效的规则
- 稳定背景信息

### 不适合进 `MEMORY.md`
- 一次性的临时琐事
- 很快过期的信息
- 没有长期价值的聊天废话

否则 MEMORY.md 很快会胖成垃圾场。

---

## 五、今天的验收标准

你做完后要能告诉我：

1. 你让 OpenClaw 记住了什么
2. 它写进了哪个文件
3. 你怎么验证它真的写进去了
4. `MEMORY.md` 和 daily memory 的区别是什么

---

## 常见坑

### 1）以为“它回复知道了”就等于记住了
错。看文件。

### 2）什么都往长期记忆里塞
久了只会越来越难用。

### 3）只写 daily，不整理长期记忆
那就会变成一堆流水账，真正有用的信息埋死在里面。
