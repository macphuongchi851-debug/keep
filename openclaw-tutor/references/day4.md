# Day 4：模型切换与验证 —— 别再靠猜，真的切一次

## 今天为什么学这个

昨天你已经知道 primary / fallback / alias 是什么。

今天开始真动手：
- 列出模型
- 切换模型
- 验证切换是不是确实生效

重点不是“发了 `/model` 就自我感动”，而是确认系统真的换了。

## 今天的目标

你需要做到：

1. 列出可用模型
2. 成功切到另一个模型
3. 用状态或对话验证切换生效
4. 知道最常见的模型切换失败原因

---

## 一、先列出模型

```bash
openclaw models list
```

如果你习惯在聊天里操作，也可以试：

```text
/models
```

记下至少 3 个模型名。

---

## 二、切换模型

### 命令行方式

```bash
openclaw models set <model-or-alias>
```

### 聊天方式

```text
/model provider/model-name
```

切换时，优先选一个你在 `models list` 里明确看到、而且当前看起来可用的模型。

---

## 三、立刻验证

切完后别靠感觉，马上验证：

```bash
openclaw models status
```

或者在对话里问：

```text
你现在用的是什么模型？
```

最好两边都交叉确认。

---

## 四、最常见的失败原因

### 1）模型名写错
尤其 provider/model 格式写错。

### 2）模型没配置/没授权
你知道这个名字，不代表你环境里真能用。

### 3）切了但没验证
很多人最大的问题不是切失败，而是压根不知道自己有没有切成功。

---

## 今天的验收标准

你做完后要给我：

1. 你看到的 3 个模型
2. 你切换成了哪个
3. 你如何确认切换生效
4. 如果切失败，错误大概是什么

---

## 做完后怎么回我

你直接发：

1. 目标模型名
2. 切换后的状态结果
3. 你对“验证”这件事的理解
