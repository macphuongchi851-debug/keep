# Day 5：Workspace 与文件操作 —— 让 OpenClaw 真正开始替你动手

## 今天为什么学这个

很多人会和 OpenClaw 聊半天，但一直停留在“会回话”。

真正拉开差距的，是它能不能：
- 读文件
- 写文件
- 改文件
- 在 workspace 里留下可追踪结果

今天这节过后，它才不只是个聊天窗口，而是开始变成你的执行型助手。

## 今天的目标

你需要做到：

1. 理解 workspace 是默认工作区
2. 让 OpenClaw 读取一个文件
3. 让 OpenClaw 创建或修改一个文件
4. 知道为什么“写进文件”比“口头记住”更可靠

---

## 一、先复习一句最重要的话

**OpenClaw 的很多“记住”和“做过”，本质上都要落到文件。**

尤其是：
- `MEMORY.md`
- `memory/YYYY-MM-DD.md`
- 你的项目文件
- 课程/脚本/配置说明文件

文件是证据，嘴不是。

---

## 二、看一下当前 workspace

先执行：

```bash
cd ~/.openclaw/workspace
pwd
ls
```

如果你想看得更完整：

```bash
find ~/.openclaw/workspace -maxdepth 2 -type f | sort
```

你要建立一个感觉：
- 哪些是人格设定
- 哪些是长期记忆
- 哪些是项目文件

---

## 三、做一个最小文件实验

### 方式 A：让 OpenClaw 帮你创建文件
你可以直接对它说：

```text
在 workspace 里帮我创建一个 test-note.md，写上今天是 Day 5 测试。
```

### 方式 B：让 OpenClaw 帮你读取文件
比如让它读取：

```text
读取 USER.md，然后告诉我里面记录了哪些用户偏好。
```

### 方式 C：让 OpenClaw 帮你追加一条 daily memory
比如：

```text
把“今天开始系统学习 OpenClaw 文件操作”记到今天的 daily memory 里。
```

---

## 四、观察结果，而不是只听它嘴上说完成了

你做完后，自己也要去看文件：

```bash
cat ~/.openclaw/workspace/test-note.md
```

或者：

```bash
cat ~/.openclaw/workspace/memory/$(date +%F).md
```

别只看聊天里一句“已完成”。

要养成习惯：
- **能验证就验证**
- 文件存在才算真干了

---

## 五、理解三种常见文件操作心智

### 1. read
读取现有内容。适合：
- 看配置
- 看文档
- 看日志片段

### 2. write
覆盖写入。适合：
- 新建文件
- 整体重写文件

### 3. edit
精确替换。适合：
- 对已有文件做局部修改
- 避免整文件重写造成误伤

## 常见坑

### 1）以为它“答应了”就等于“写成功了”
错。去看文件。

### 2）把长期记忆和临时聊天混在一起
要记住的内容，要写入 memory / MEMORY，不是留在聊天里等失忆。

### 3）不理解覆盖和局部编辑的区别
粗暴重写很容易把好东西一起干没。
