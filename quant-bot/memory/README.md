# quant-bot/memory/README.md

> **用途**：说明 Quant Bot 的记忆该怎么分流。
> **核心原则**：量化分析、市场研究、策略框架留本地；跨 bot 通用的规则和环境经验再上升到 `../shared/`。

## 记到本地 `memory/` 的内容

适合写进 `quant-bot/memory/`：
- 当天行情分析过程
- 某次策略思路、验证过程、研究笔记
- 只对 Quant Bot 有用的市场框架
- 量化研究、风险分析、复盘专题

## 记到 `quant-bot/MEMORY.md` 的内容

适合写：
- 长期有效的量化分析框架
- 稳定的研究输出规范
- 常用风险约束
- 本 bot 的专题索引

## 应上升到 `../shared/` 的内容

如果满足“多个 bot 都可能复用”，就上 shared：
- 用户通用偏好
- 通用执行纪律
- 通用环境 / 代理 / 容器 / 权限经验
- 通用术语 / 简称

## 不该记的内容

- 一次性市场噪音
- 没有复用价值的临时判断
- 其他 bot 的业务流程
- 其他 bot 的人格设定

## 一句话判断

> 量化研究和策略经验留本地；跨 bot 共通知识上 shared；当天分析过程先进 daily。
