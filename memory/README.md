# memory/README.md

> **用途**：说明 memory 目录里每个文件各自负责什么，以及一条信息该写到哪里。
> **什么时候看**：当你准备记东西、整理记忆、复盘资料，或者不确定该写进哪个文件时。
> **核心原则**：先判断这条信息是“当天过程”、"长期规则"、"项目经验"、"环境排障结论"，再决定落点。

---

## 先记住最简单的判断法

问自己 4 个问题：

1. **这只是今天这次过程里有用吗？**
2. **这是不是长期稳定有效的规则/偏好？**
3. **这是不是某个项目或方案的可复用经验？**
4. **这是不是环境、权限、代理、容器、部署类排障结论？**

按答案分流：

- **只对今天这次过程有用** → `memory/YYYY-MM-DD.md`
- **长期稳定规则 / 偏好 / 索引** → `MEMORY.md`
- **项目打法 / 方案沉淀** → `memory/project-experience.md`
- **环境 / 排障 / 部署经验** → `memory/environment-notes.md`
- **你我之间的沟通规矩** → `memory/rules.md`

如果都不满足，且以后大概率不会再查：
- **不值得记**

## 各文件怎么用

### 1) `memory/YYYY-MM-DD.md`

**适合写：**
- 今天发生了什么
- 某次排障过程
- 今天试了哪些方案
- 阶段性结果
- 今天学了什么

**不适合写：**
- 长期稳定偏好
- 已经验证很多次的固定规则
- 项目总方案总结

**一句话理解：**
> 这是当天日志，不是长期结论库。

---

### 2) `MEMORY.md`

**适合写：**
- 用户长期偏好
- 稳定执行纪律
- 高频复用的重要约定
- 专题文件索引

**不适合写：**
- 细碎过程
- 很长的项目细节
- 大段环境排障步骤

**一句话理解：**
> 这是总索引 + 最重要的长期记忆，必须轻。

---

### 3) `memory/project-experience.md`

**适合写：**
- 某个项目的默认打法
- 某类方案的固定结构
- 已验证有效的执行模板
- 以后提到某项目就能直接复用的结论

**不适合写：**
- 临时试错过程
- 纯沟通偏好
- 纯环境配置问题

**一句话理解：**
> 这是“项目脑子”。

---

### 4) `memory/environment-notes.md`

**适合写：**
- 权限问题怎么处理
- 沙箱 / 容器 / 代理 / 网络问题怎么排
- 某类环境报错的优先修法
- 关键命令或修复方式

**不适合写：**
- 用户偏好
- 项目内容策划
- 一次性闲聊内容

**一句话理解：**
> 这是“环境和排障经验库”。

---

### 5) `memory/rules.md`

**适合写：**
- 用户怎么希望被称呼
- 回复风格要求
- 做事反馈纪律
- 配置修改前后的硬规矩
- 长期有效的沟通边界

**不适合写：**
- 项目具体方案
- 环境故障细节
- 单日任务过程

**一句话理解：**
> 这是“你我合作方式说明书”。

---

## 一张超短判断表

- **今天发生的过程** → `memory/YYYY-MM-DD.md`
- **长期偏好 / 规则 / 索引** → `MEMORY.md`
- **项目沉淀 / 固定打法** → `memory/project-experience.md`
- **环境 / 容器 / 代理 / 排障** → `memory/environment-notes.md`
- **沟通规矩 / 执行纪律** → `memory/rules.md`
- **一次性碎信息** → 不记

---

## 例子

### 例子 1
“今天修 Telegram，发现是 allowlist 填错了 ID。”

怎么记：
- 先记 `memory/YYYY-MM-DD.md`
- 如果以后很容易重复踩坑，再提炼进 `memory/environment-notes.md`

### 例子 2
“用户要求以后默认中文、回复短一点。”

怎么记：
- 记 `MEMORY.md`
- 也可同步整理进 `memory/rules.md`

### 例子 3
“以后提到 A8Ads 那套，默认就是 42 秒横版宣传片方案。”

怎么记：
- 记 `memory/project-experience.md`
- `MEMORY.md` 只保留索引，不重复堆细节

### 例子 4
“Chromium 容器外网访问失败时，优先加 HTTP_PROXY / HTTPS_PROXY / NO_PROXY。”

怎么记：
- 记 `memory/environment-notes.md`

### 例子 5
“今天上海 18°C，我建议穿薄外套。”

怎么记：
- 通常**不记**
- 如果它只对今天流程有一点上下文价值，也最多记 daily，不进长期

---

## 最后一句话

判断记哪儿时，别想太玄：

> **当天过程进 daily，长期规则进 MEMORY，项目经验进 project-experience，环境排障进 environment-notes，合作规矩进 rules。**

再不确定，就先记 daily，后面再提炼。这个最稳。
