# memory/templates.md

> **用途**：提供统一的记忆录入模板，减少临时发挥，方便后续整理和提炼。
> **什么时候用**：当你准备往 daily、长期记忆、项目经验、环境排障、沟通规矩里新增内容时。
> **怎么用**：复制对应模板，删掉不需要的字段，填入真实内容即可。

---

## 1) Daily 模板（`memory/YYYY-MM-DD.md`）

**适用场景：**
- 今天发生了什么
- 某次排障过程
- 阶段性结果
- 临时上下文

### 模板

```md
# YYYY-MM-DD

- 事项：
- 背景：
- 我做了什么：
- 结果：
- 卡点 / 问题：
- 后续是否值得提炼进长期记忆：是 / 否
- 如果值得，准备提炼到：`MEMORY.md` / `memory/project-experience.md` / `memory/environment-notes.md` / `memory/rules.md`
```

### 极简版

```md
- 事项：
- 结果：
- 是否值得提炼长期：
```

## 2) 长期记忆模板（`MEMORY.md`）

**适用场景：**
- 用户长期偏好
- 稳定执行纪律
- 高频复用约定
- 总索引项

### 模板

```md
- 主题：
- 类型：用户偏好 / 执行纪律 / 长期约定 / 专题索引
- 内容：
- 生效范围：默认全局 / 某项目 / 某渠道
- 为什么值得长期保留：
```

### 极简版

```md
- 类型：
- 内容：
```

## 3) 项目经验模板（`memory/project-experience.md`）

**适用场景：**
- 某项目默认打法
- 某类方案固定结构
- 已验证有效的执行模板
- 以后提到项目名就能直接复用的结论

### 模板

```md
## 项目 / 主题名

- 背景：
- 默认触发词：
- 默认方案 / 结构：
- 核心卖点 / 目标：
- 已验证有效的做法：
- 不建议做的内容：
- 可直接复用的素材 / 文件：
```

### 极简版

```md
## 项目 / 主题名
- 默认方案：
- 可复用结论：
```

## 6) 一条信息不知道该写哪时，用这个判断模板

```md
- 这条信息是什么：
- 它是当天过程，还是长期结论：
- 它更像用户偏好 / 项目经验 / 环境排障 / 沟通规矩中的哪一类：
- 建议落点：
- 如果暂时拿不准：先记到 `memory/YYYY-MM-DD.md`
```

## 7) 最后给自己的一句检查

写之前问一句：

> **这条内容以后还会被翻出来用吗？**

- 会，而且是长期稳定的 → 进长期相关文件
- 可能会，但主要是今天过程 → 先进 daily
- 不会 → 别记，别制造垃圾
