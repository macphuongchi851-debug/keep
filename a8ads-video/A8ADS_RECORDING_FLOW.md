# A8Ads 录制脚本与执行流程（优化版）

## 目标
把 A8Ads 产品演示录成一条**稳定、可重复、适合后续剪辑**的横版素材。

核心原则：
1. **录制和动作分离**：先用固定参数录屏，再执行页面动作。
2. **自动正常收尾**：严禁靠强杀 ffmpeg 结束，否则 mp4 容易坏。
3. **镜头脚本固定**：每一段做什么、停多久、滚多远，尽量写死。
4. **参数化**：后续只调时长、帧率、压缩率，不重新拼命令。

---

## 脚本文件
统一录制器：

```bash
scripts/a8ads_recorder.sh OUTPUT.mp4 [duration_seconds]
```

默认参数：
- 时长：92 秒
- 帧率：15 fps
- 缩放宽度：1600
- CRF：31
- preset：veryfast
- 自动 faststart
- 无音频

可选环境变量：
- `WINDOW_PATTERN`
- `FPS`
- `SCALE_WIDTH`
- `CRF`
- `PRESET`
- `DISPLAY_NUM`
- `XAUTHORITY_PATH`

例子：

```bash
FPS=20 CRF=29 scripts/a8ads_recorder.sh a8ads_video_assets/take.mp4 95
```

## 推荐正式版录制参数
### Telegram 可直接回传版
适合先看效果：

```bash
scripts/a8ads_recorder.sh a8ads_video_assets/full_flow_telegram.mp4 92
```

特点：
- 文件更小
- 易回传
- 适合测试节奏和流程

### 本地保真版
适合后续剪辑：

```bash
FPS=20 SCALE_WIDTH=1920 CRF=24 PRESET=medium scripts/a8ads_recorder.sh a8ads_video_assets/full_flow_master.mp4 92
```

特点：
- 更清晰
- 更适合二次剪辑
- 文件会明显更大

## 推荐执行流程（固定镜头脚本）
总时长建议：**约 90–95 秒原始录屏**。
后期剪成 30–45 秒。

### Scene 1：首页开场（0–12s）
目标：建立产品感。

动作：
1. 打开首页 `https://www.a8ads.com/`
2. 停 2 秒
3. 缓慢下滚到 220
4. 缓慢下滚到 520
5. 回滚到 260
6. 回到顶部

建议：
- 首页动作不要太猛，像“展示”不是“找东西”
- 给后期留 2–3 个可切镜头点

---

### Scene 2：广告库搜索（12–34s）
目标：突出核心卖点。

动作：
1. 打开 `https://www.a8ads.com/ads`
2. 等待 2.5–3 秒
3. 搜索关键词 `dog`
4. 等待结果出现
5. 缓慢滚到 320
6. 缓慢滚到 760
7. 回到 380
8. 停在 120 左右

建议：
- 这是整条片子的主镜头
- 结果出现后至少停 1 秒
- 滚动幅度不要过大，避免像乱翻页

---

### Scene 3：广告主页（34–50s）
目标：展示广告主维度。

动作：
1. 打开 `https://www.a8ads.com/advertisers`
2. 停 2 秒
3. 滚到 220
4. 滚到 520
5. 回到 240
6. 回顶部

建议：
- 保留“看列表”的停留感
- 不用做搜索，避免信息太碎

---

### Scene 4：产品库（50–66s）
目标：形成“广告 → 产品”的逻辑链。

动作：
1. 打开 `https://www.a8ads.com/products`
2. 停 2 秒
3. 滚到 260
4. 滚到 560
5. 回到 300
6. 回顶部

建议：
- 停留要比广告主页面略短
- 保留给剪辑做“功能切换感”

---

### Scene 5：套餐页（66–80s）
目标：做商业化收口。

动作：
1. 打开 `https://www.a8ads.com/packages`
2. 停 2 秒
3. 滚到 180
4. 滚到 420
5. 回到 120
6. 回顶部

建议：
- 套餐页不要占太长
- 这段更多是“补信息”，不是绝对主卖点

---

### Scene 6：回首页收尾（80–92s）
目标：给剪辑留结尾空间。

动作：
1. 回首页
2. 停 4–5 秒
3. 不再大幅滚动

建议：
- 结尾尽量稳
- 给字幕/LOGO/CTA 留位置

---

## 实操顺序（每次录制都按这个来）
1. 确认 Chrome 已打开 A8Ads 标签页，并且窗口可见
2. 启动录制脚本（固定时长）
3. 立刻按镜头脚本执行页面动作
4. 不手动终止 ffmpeg
5. 等待脚本自己封装完成
6. 检查文件大小与可播放性
7. 再决定是否回传 Telegram，或保留本地 master

## 后续优化建议
### 如果想更像宣传片
优先调这些，不要乱改总体结构：
- 每段停留时长
- 滚动幅度
- 页面顺序
- 搜索关键词

### 如果想更适合剪辑
建议每段单独录一个 clip：
- 01_home.mp4
- 02_ads_search.mp4
- 03_advertisers.mp4
- 04_products.mp4
- 05_packages.mp4
- 06_home_end.mp4

这样后期会比一整条 master 更好剪。

---

## 当前结论
相比旧方案，这版优化点：
- 去掉了对 `xdotool` 的依赖
- 统一了 ffmpeg 入口
- 自动收尾，降低 mp4 损坏概率
- 参数可调，方便按“Telegram 版 / Master 版”分开输出
- 录制流程更固定，不容易临场乱掉
